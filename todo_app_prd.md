# Todo App PRD

## 1. 한 줄 정의

**가장 하기 싫은 일을 가장 먼저 시작하게 만들고, 흩어진 메모·회의록·아이디어를 AI가 실행 가능한 Todo로 정리해주되, 최종 반영은 항상 사용자가 승인하는 생산성 앱**

---

## 2. 제품 목표

이 앱의 목표는 단순한 할 일 기록이 아니다.

1. **회피하는 일을 먼저 하게 만드는 것**
2. **생각, 회의록, 메모를 실행 가능한 액션으로 바꾸는 것**
3. **AI가 정리해주되 사용자가 통제권을 잃지 않게 하는 것**
4. **정리되지 않은 정보와 실행 리스트를 하나의 흐름으로 연결하는 것**

---

## 3. 해결하려는 문제

### 문제 1. 가장 중요한 일이 아니라, 쉬운 일부터 하게 된다
사용자는 종종 가장 부담스럽고 하기 싫은 일을 미루고, 대신 쉬운 일을 먼저 처리한다.  
그 결과 바쁜데도 핵심 진척은 없다.

### 문제 2. Todo와 메모가 분리되어 있다
실제 업무는 Todo만으로 이루어지지 않는다.  
회의록, 아이디어, 고민거리, 확인 필요 사항이 같이 쌓이는데 대부분 별도 앱이나 문서에 흩어진다.

### 문제 3. 메모에서 실행 항목이 자동으로 연결되지 않는다
회의록이나 아이디어를 적고 나서 다시 사람이 Action Item을 추출하고 Todo를 등록해야 한다.  
이 과정이 번거로워서 실행이 누락된다.

### 문제 4. AI 자동화는 편하지만, 원치 않는 변경이 불안하다
AI가 할 일을 멋대로 추가하거나 수정하면 신뢰가 떨어진다.  
사용자는 **추천은 원하지만 최종 결정권은 본인이 갖고 싶다.**

---

## 4. 타겟 사용자

### 1차 타겟
- 해야 할 일은 많은데 우선순위 실행이 잘 안 되는 직장인
- 회의/메모/아이디어가 많고 이를 실행으로 연결하고 싶은 사람
- AI의 도움은 받고 싶지만 자동 반영은 부담스러운 사용자

### 2차 타겟
- 개인 프로젝트를 관리하는 1인 창작자/기획자/개발자
- 업무 단위로 큰 Task 아래 세부 Todo를 관리하고 싶은 사용자

---

## 5. 핵심 가치 제안

### 5-1. “가장 하기 싫은 것부터”
앱은 단순히 우선순위를 보여주는 것이 아니라,  
사용자가 회피하는 일을 **가장 먼저 시작하도록 심리적으로 유도**한다.

예:
- 오늘 가장 피하고 있는 일
- 이 일을 먼저 끝내면 나머지는 쉬워집니다
- 10분만 시작해보세요
- 가장 무거운 일 먼저 치우기

### 5-2. Todo + Notes + AI + Approval
각 Task 안에 다음이 함께 존재한다.
- Todo
- 회의록/메모/아이디어
- AI 요약
- 추가 확인 포인트
- 고민 포인트
- Action Item 제안
- Todo 생성 제안

단, **AI가 제안만 하고 실제 생성/반영은 사용자가 승인**한다.

### 5-3. 언제든 다시 처리 가능한 “보관함”
AI가 뽑아낸 제안(Action Item, Todo 생성안, 확인 필요사항)은 즉시 반영되지 않아도 된다.  
사용자는 **보관함(Inbox/Review Queue)** 에서 나중에 검토 후 처리할 수 있다.

---

## 6. 제품 원칙

1. **AI는 자동 실행보다 보조와 정리에 집중한다**
2. **실제 데이터 변경은 사용자 승인 후에만 일어난다**
3. **입력은 자유롭게, 결과는 구조화되게**
4. **큰 일(Task)과 작은 실행(Todo)을 분리한다**
5. **회피하는 일에 대한 시작 유도 경험이 핵심 차별점이다**

---

## 7. 주요 개념 정의

### 7-1. Task
하나의 큰 업무 단위.  
예: “주간 보고 준비”, “이직 준비”, “집 계약 준비”

포함 요소:
- Task 제목
- 설명
- 상태
- 마감일
- 연결된 Todo 목록
- 연결된 노트/회의록/아이디어
- AI 요약
- AI 제안 보관함

### 7-2. Todo
실제 실행 가능한 최소 단위 작업.  
예: “보고서 목차 작성”, “부동산에 전화”, “면접 예상질문 정리”

포함 요소:
- 제목
- 상태(미시작/진행중/완료)
- 중요도
- 하기 싫음 정도(선택)
- 예상 소요시간
- 상위 Task

### 7-3. Note
자유 입력 데이터.
- 회의록
- 메모
- 아이디어
- 고민 정리
- 음성 기록 텍스트 변환본(향후 확장 가능)

### 7-4. AI Suggestion
AI가 Note를 분석해 만든 구조화 결과.
- 요약
- 추가 확인 필요 사항
- 고민/의사결정 필요 사항
- Action Item 후보
- Todo 생성 후보

### 7-5. Archive / Review Queue
AI 제안이 저장되는 검토함.
사용자가 여기서:
- 승인
- 수정 후 승인
- 보류
- 삭제
를 할 수 있다.

## 8. 핵심 사용자 시나리오

### 시나리오 1. 가장 하기 싫은 일을 먼저 하게 만들기
1. 사용자가 오늘의 Todo 목록을 연다
2. 각 Todo에 중요도/부담도/하기 싫음 정도가 반영된다
3. 앱은 “가장 하기 싫지만 중요한 일”을 상단에 강조한다
4. 사용자는 “10분만 시작하기” 버튼으로 진입한다
5. 시작하면 해당 Todo는 진행중으로 변경되고, 앱은 짧은 독려 메시지를 준다

### 시나리오 2. Task 아래에 자유롭게 회의록 입력
1. 사용자가 특정 Task에 들어간다
2. 회의 내용/아이디어/걱정되는 부분을 자유롭게 적는다
3. 저장 시 원문은 그대로 보관된다
4. AI가 이를 분석해 요약과 Action Item 후보를 생성한다
5. 생성된 후보는 즉시 Todo에 반영되지 않고 Review Queue에 들어간다
6. 사용자가 검토 후 승인하면 Todo가 생성된다

### 시나리오 3. 나중에 보관함에서 일괄 처리
1. 사용자는 바쁜 상황에서 일단 Note만 남긴다
2. AI가 여러 건의 제안을 만든다
3. 사용자는 나중에 보관함에서 모아서 본다
4. 각 제안에 대해 승인/수정/삭제를 한다
5. 승인된 것만 실제 Task/Todo에 반영된다

## 9. 기능 요구사항

### 9-1. Todo 관리
- Todo 생성/수정/완료/삭제
- Todo를 Task에 연결
- 중요도 설정
- 예상 소요시간 설정
- 하기 싫음 정도 설정
- 오늘 할 일 보기
- 미완료 Todo 다시 제안

### 9-2. Task 관리
- Task 생성/수정/삭제
- Task별 Todo 묶음 보기
- Task별 Note/회의록/아이디어 저장
- Task 상태 관리
- Task 단위 진행률 표시

### 9-3. Note 입력 및 보관
- 자유 텍스트 입력
- 회의록/아이디어/메모 유형 선택
- 원문 보관
- Note를 특정 Task에 연결
- Note 작성 시점/작성자/태그 저장

### 9-4. AI 분석
사용자가 Note를 저장하면 AI는 아래를 생성한다.
- 3줄 요약
- 핵심 논의 사항
- 추가 확인해야 하는 부분
- 더 고민해야 하는 부분
- Action Item 후보
- Todo 후보
- 관련 기존 Todo/Task와의 연결 제안

### 9-5. AI 제안 검토 흐름
- AI가 생성한 결과는 “제안 상태”로 저장
- 사용자는 검토 화면에서 항목별 승인 가능
- Todo 생성은 개별 승인 가능
- 일괄 승인/일괄 삭제 가능
- 승인 전에 텍스트 수정 가능
- 승인 이력 저장

### 9-6. 보관함
- 승인 대기중 제안 목록
- Task별/날짜별/유형별 필터
- 오래된 미처리 제안 리마인드
- 보류 상태 지원
- “나중에 보기” 기능

### 9-7. 하기 싫은 일 우선 실행 유도
앱의 차별 기능.
- Todo 생성 시 “하기 싫음 정도” 선택
- 중요도 + 마감 + 하기 싫음 정도 기반으로 추천
- 홈에서 “오늘 가장 먼저 해야 할 일” 1개 강조
- 시작 유도 문구 제공
- “10분만 시작”, “첫 단계만 하기”, “작게 쪼개기” 제공
- 오래 미룬 핵심 Todo 재추천

## 10. 우선순위 로직 초안

앱은 단순 중요도순이 아니라 아래 기준을 혼합한다.

### 추천 점수 후보 요소
- 중요도
- 마감 임박도
- 하기 싫음 정도
- 미뤄진 횟수
- 예상 소요시간
- 상위 Task의 중요도

### 추천 규칙 예시
- 중요도 높고 하기 싫음 높으면 최상단 추천
- 마감 임박 + 미뤄진 횟수 높으면 경고성 노출
- 큰 부담 작업은 “바로 완료”보다 “시작” 중심 CTA 제공

## 13. 비기능 요구사항

- 원문 Note는 항상 보관되어야 함
- AI 결과와 원문은 구분 저장되어야 함
- AI 제안은 사용자 승인 전 실제 Todo에 반영되면 안 됨
- 승인/수정/삭제 이력이 남아야 함
- 응답 속도:
  - 일반 화면 조회: 1초 이내 체감
  - AI 분석: 비동기 처리 허용, 분석 완료 시 표시
- 개인정보/민감정보가 포함될 수 있으므로 저장 및 AI 처리 범위 고지 필요

## 17. 리스크

### 1. AI 제안 품질이 낮으면 신뢰가 떨어짐
대응:
- 자동 반영 금지
- 원문과 제안을 나란히 비교 제공
- 수정 후 승인 가능하게 설계

### 2. 너무 많은 제안이 쌓이면 피로해짐
대응:
- 중요도 높은 제안 우선 노출
- 일괄 처리 기능
- 오래된 제안 정리 UX 제공

### 3. “하기 싫은 일 먼저”가 스트레스로 느껴질 수 있음
대응:
- 강제가 아닌 추천
- “작게 시작하기” 옵션 제공
- 사용자가 추천 강도 조절 가능

## 19. MVP 한 줄 정리

**큰 Task 아래에 Todo와 Note를 함께 관리하고, Note에서 AI가 실행 항목을 추출해주되, 사용자가 승인한 것만 실제 Todo로 반영되는 앱. 그리고 매일 가장 하기 싫지만 중요한 일을 가장 먼저 시작하도록 유도한다.**
